🧊 3D建模方案 & 可行性评估

方案A：Mapbox GL 3D（推荐）

费用：Free tier 50,000次加载/月

实现：Mapbox GL JS + 3D地形 + 3D建筑

效果：有3D地形起伏、建筑立体、光影效果

开发量：2-3天接入

古蔺适配：下载古蔺县DEM地形数据 → 导入Mapbox tileset

方案B：Three.js + OSM数据

费用：¥0，纯开源

实现：Three.js加载OSM建筑+道路数据

效果：可做纯3D场景，但需自己写渲染器

开发量：1-2周

古蔺适配：已有OSM数据，需加入高程DEM

方案C：Cesium.js（最炫）

费用：免费版无水印

实现：Cesium 3D地球 + 古蔺县区域突出显示

效果：卫星图+3D地形+飞行视角，视觉效果最好

开发量：1周

缺点：加载量大，手机上可能卡

方案D：当前2.5D升级

费用：¥0

实现：在现有Leaflet基础上叠加3D建筑标记

效果：伪3D，每个建筑画不同高度的柱体

开发量：2-3天

优点：兼容手机、加载快、迭代快

📌 推荐路线：先走方案D，再升级方案A

Phase 1（当前v0.3）：Leaflet 2D地图 + 闯关游戏 ← 你现在看到的就是
Phase 2（1周后）：    给建筑加3D柱体，显示高度差 ← 方案D
Phase 3（1个月后）：  接入Mapbox GL，真3D地形 ← 方案A
Phase 4（远期）：     接Cesium做飞行导览 ← 方案C

3D建模难点：
- 古蔺县是山地地形，高程数据需要SRTM/DEM（免费可下载）
- OSM缺少建筑高度数据 → 需要用AI估算或实地采集
- 手机3D渲染性能有限 → 需要做LOD层级优化
- 国内访问Mapbox/Cesium CDN可能慢 → 需要做备选方案
